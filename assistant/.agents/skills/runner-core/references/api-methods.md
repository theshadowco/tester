# Методы API Runner

Этот документ демонстрирует варианты использования методов для навыка `runner-core`.

## Общие правила параметров

Многие методы Runner принимают параметр `Имя`.

- `Имя` может быть идентификатором элемента управления или заголовком.
- Префикс `#` перед `Имя`, указывает Runner-у на, что элемент в интерфейсе следует искать по внутреннему идентификатору.
- Поддерживаются полные пути, например `#Товары / #Сумма` или `#Дополнительно / #Код`.
- Перед взаимодействием с элементом управления убедись, что он существует в текущей форме, через `ПолучитьЭлементыАктивногоОкна` (если ты только что открыл окно), или через `ПолучитьИзмененияОкна` (если ты для этого окна уже вызывал `ПолучитьЭлементыАктивногоОкна`, но успел что-то в нём изменить).

## Подготовка сессии и контекст формы

### `Подключить`

```bsl
Подключить();
```

Вызывай этот метод перед любым взаимодействием с приложением.

### `Здесь(Заголовок = неопределено)`

```bsl
Здесь();
Здесь("Товары (список)");
Здесь("Заказ покупателя (созд*");
```

Переводит форму или окно в контекст выполнения скрипта. Используй этот метод перед любым взаимодействием формой.

- `Заголовок` необязателен, поддерживаются маски `*` и `?`. Если заголовок не указан - контекст выполнения будет установлен для текущей активной формы.

### `Дождаться(Заголовок, Таймаут = 3)`

```bsl
если (не Дождаться("Мастер*", 5)) тогда
    // ... действия в случае, когда окно мастера ещё не появилось;
конецесли;
```

Ждёт появления формы или окна.

- `Заголовок` - заголовок формы, поддерживаются маски `*` и `?`.
- Возвращает `истина`, если окно обнаружено, иначе `ложь`.

### `Закрыть(Заголовок = неопределено)`

```bsl
Закрыть();
Закрыть("Заказ покупателя *");
```

Нажимает на кнопку закрытия окна по переданному заголовку. Если в форме есть не сохранённые данные, система может дополнительно запросить подтверждение пользователя (которое никак не обрабатывается этим методом).

- `Заголовок` - заголовок формы, поддерживаются маски `*` и `?`. Если заголовок не указан, будет закрыта текущая активная форма.

### `ЗакрытьВсё()`

```bsl
ЗакрытьВсё();
```
Закрывает все окна в приложении, при этом Runner будет пытаться отвечать `Нет` на запросы сохранения не сохранённых данных.

## Просмотр и чтение

### `ПолучитьГлавноеМеню()`

```bsl
return ПолучитьГлавноеМеню();
```

Возвращает структуру главного меню как массив подсистем и элементов с навигационными к ним ссылками. Используй этот метод когда ты еще не знаешь функциональность приложения на самом верхнем уровне.

### `ПолучитьЭлементыАктивногоОкна()`

```bsl
возврат ПолучитьЭлементыАктивногоОкна();
```

Возвращает дерево элементов управления активного окна. Используй этот метод, чтобы полностью осмотреть всю форму. Впоследствии, если ты что-то меняешь на форме, используй метод `ПолучитьИзмененияОкна`. Таким образом, ты будешь получать только изменения по сравнению с последним состоянием формы, а не всю форму каждый раз.

### `ПолучитьИзмененияОкна()`

```bsl
return ПолучитьИзмененияОкна;
```

Возвращает только изменения в активном окне с момента последнего получения информации методом `ПолучитьЭлементыАктивногоОкна`.

Ожидаемое использование: для вновь открытой формы, один раз вызови `ПолучитьЭлементыАктивногоОкна`, затем, после каждого действия, изменяющего форму, вызывай `ПолучитьИзмененияОкна`, чтобы существенно уменьшить объём информации для анализа. Возвращаемые изменения должны быть достаточны, чтобы обнаружить как минимум:
- добавленные/удаленные элементы управления
- измененные значения
- измененную видимость / доступность / заголовки
- если активное окно полностью изменилось и компактная дельта ненадежна, можно заново вызвать `ПолучитьЭлементыАктивногоОкна`.

### `Взять(Имя)`

```bsl
код = Взять("#Дополнительно / #Код");
текст = Взять("#ОтчётТабДок [R1C1]");
количество = Взять("#ТоварыКоличество [ 2 ]");
```

Возвращает строковое представление значения в элементе формы. Возвращаемые значения следуют правилам форматирования и региональным настройкам информационной базы.

### `ПолучитьСодержимоеТаблицы(Имя)`

```bsl
строки = ПолучитьСодержимоеТаблицы("#Товары");
возврат строки;
```

Возвращает содержимое таблицы как массив строк. Если на момент получения вызова этого метода, таблица находилась в режиме редактирования, Runner завершает редактирование и начинает чтение таблицы с первой строки.

- Метод читает до 100 строк. Для больших таблиц, сначала сузь список при помощи стандартного механизма поиска/отбора данных для табличных частей, или используй экспорт таблицы в табличный документ для дальнейшего анализа методом `ПолучитьСодержимоеТабличногоДокумента`.
- В возвращаемых значениях ячеек удаляются неразрывные пробелы.

### `ПолучитьСодержимоеТабличногоДокумента(Имя)`

```bsl
путьКФайлу = ПолучитьСодержимоеТабличногоДокумента("#ОтчётРезультат");
возврат путьКФайлу;
```

Экспортирует поле табличного документа во временный файл `.xlsx`. Метод выбрасывает исключение, если целевой элемент управления не является полем табличного документа.

### `ПолучитьСнимок()`

```bsl
возврат ПолучитьСнимок();
```

Возвращает путь к временному PNG-файлу со снимком экрана подключённого приложения. Используй только когда дерево элементов управления недостаточно хорошо объясняет состояние UI.

## Навигация и взаимодействие

### `Коммандос(НавигационнаяСсылка)`

```bsl
Коммандос("e1cib/list/Справочник.Изделия");
```

Выполняет системную навигационную команду. После открытия формы `Коммандос` неявно вызывает `Здесь` для активации контекста формы, к которой был совершён переход. Используй этот метод только когда навигационная ссылка уже известна и была ранее получена из подключённого приложения.

### `Нажать(Имя)`

```bsl
Нажать("#ФормаСохранить");
Нажать("#Links[2]");
Нажать("#Links[Поступление товаров №123456 dated 01\\/01\\/2019]");
```

Активирует и нажимает элемент управления. Применимо к кнопкам, декоративным элементам, полям и похожим кликабельным элементам управления. После нажатия, не забывай осматривать изменения состояния UI.

### `ВнестиЗначение(Имя, Значение)`

```bsl
ВнестиЗначение("#Комментарий", "Важный комментарий");
ВнестиЗначение("#ТоварыКоличество [ 2 ]", 15);
ВнестиЗначение("#РезультатТабДок [R1C1]", "Привет, как дела?");
```

Активирует поле и вводит туда значение. Этот метод может изменять состояние формы, так как после изменения значения в поле, может быть запущена программная логика, которая может изменять другие поля или открывать другие диалоги, списки или формы. Перед следующим действием, получай новое состояние через `ПолучитьИзмененияОкна`.

### `Очистить(Имя)`

```bsl
Очистить("#СписокСтатус");
```

Очищает целевое поле. По смыслу, метод `Очистить` аналогичен `ВнестиЗначение`, поэтому не забывай вызывать `ПолучитьИзмененияОкна` перед следующим шагом.

### `Выбрать(Имя)`

```bsl
Выбрать("#Покупатель");
Выбрать("#ТоварыТовар [ 3 ]");
```

Инициирует выбор значения в поле ввода. Обычно используется, чтобы открыть форму выбора для поля ссылочного типа. Для составных типов полей, система может открыть промежуточный диалог `Выберите тип данных`:

```bsl
Здесь("Выберите тип данных");
КСтроке("#TypeTree", "", "Товары");
Нажать("#OK");
возврат ПолучитьИзмененияОкна();
```

Рассматривай этот диалог как смену формы и осматривай его перед продолжением.

### `Далее()`

```bsl
Далее();
```

Переходит к следующему полю согласно компоновке формы. Рассматривай `Далее` как потенциально изменяющий состояние, когда выход из поля запускает проверку или авто-заполнения.

## Таблицы и ожидание

### `КСтроке(Таблица, Колонка, Значение)`

```bsl
КСтроке("#Товары", "№", 3);
КСтроке("#Папки", "Имя", "Заметки");
```

Переходит к строке по точному значению в указанной колонке. В некоторых формах, переход к строке может сопровождаться обновлением формы, например, это может быть обновление данных в связанной таблице, в табличном документе или других элементов управления. Таким образом, если те предполагаешь что форма устроена именно таким образом - не забывай использовать `Пауза` и `ПолучитьИзмененияОкна`, например так:

```bsl
Подключить ();
Здесь();
КСтроке("#Отчёты", "Имя", "Декларация по НДС");
Пауза(3); // Даём время обновиться данным на форме
возврат ПолучитьИзмененияОкна ();
```
### `Пауза(Секунды)`

```bsl
Пауза(5);
```

Приостанавливает выполнение скрипта на указанное количество секунд. Используй умеренно, главным образом для связанных таблиц или медленных переходов UI, когда нет лучшего способа синхронизации.
